## 1. 项目概述

Tank War World 是一款**在浏览器里运行的方块风格 3D 多人坦克战争游戏**。玩家通过 WebSocket 连入一台权威服务器，在服务器随机生成的世界中以红/蓝两阵营对抗。核心体验由以下支柱组成：

- **方块、非精细**的美术风格（low-poly 硬表面着色，接近 flat-shaded）；
- **双视角**：第三人称跟随视角用于机动和战场感知，第一人称瞄准镜视角用于远距离精确射击；
- **半真实坦克手感**：履带式移动、惯性、质量感（基于 Jolt Physics）；
- **部位化伤害 + 功能损坏**：坦克的车身/炮塔/履带/引擎等部位各有独立子血量，被击毁会导致"无法移动"、"无法开火"等功能性后果；
- **可破坏环境**：石块、树木等障碍物也有血量，承伤达到阈值会播放损毁动画后消失；
- **抛物线弹道**：炮弹受重力影响、有飞行时间，远距离需要预判——这是瞄准镜视角存在的根本原因；
- **持续世界 + 周期重置**：服务器启动后世界持续运行，玩家随时加入/退出；达到时长上限或积分阈值时自动重置并重新生成世界。

## 3. 会话生命周期

### 3.1 世界实例

每个服务器进程承载**一个**世界实例（1km × 1km）。多实例通过启动多个进程实现（不在本 spec 范围内）。

### 3.2 匹配流程

1. 客户端连 WebSocket → 发 `CONNECT` 消息（携带玩家名、偏好阵营）；
2. 服务端分配阵营（优先填平衡；若差 2 人以上强制加入人少方）；
3. 服务端回 `CONNECT_ACK`：玩家 ID、阵营、当前世界种子、当前世界状态快照、重置倒计时；
4. 客户端进入"选择坦克 / 出生点"界面：从 3 种坦克（轻/中/重）中选一辆，从本阵营 3 个出生点中选一个；
5. 玩家按"出击"后，服务端生成坦克实体并广播。

### 3.3 重置触发（混合条件）

两个条件**任一达成**即触发世界重置：

- **时长上限**：服务器启动起 2 小时；
- **积分阈值**：任一阵营累计积分达到 **1000**。

**重置流程：**
1. 服务端广播 `WORLD_RESET_WARNING`（剩余 30s 倒计时，客户端显示在 HUD 顶部）；
2. 倒计时归零，服务端生成新世界种子、重建地形和障碍物、重置积分、所有在线玩家回到"选择坦克 / 出生点"界面；
3. 广播 `WORLD_RESET_DONE` 附带新种子和世界快照。

### 3.4 积分来源

| 事件                          | 积分  |
|-------------------------------|------:|
| 击杀敌方坦克                  | +10   |
| 助攻（60s 内先造成伤害）      | +3    |
| 摧毁敌方出生点附近占领点*     | +20   |
| 摧毁障碍物                    | 0     |

*出生点附近占领点：每个阵营基地有 1 个中央"指挥部"物件（血量较高的特殊障碍物），摧毁它给对方大量积分并短暂禁用该出生点 60s。

### 3.5 死亡重生

- 坦克整车血量归 0 或关键部位（hull）完全损毁 → 播放爆炸动画、坦克残骸保留 15s 后消失；
- 玩家进入重生界面：固定 **10 秒冷却**；
- 冷却期间可从本阵营 **3 个预置出生区域**中选择一个；
- 可在重生时选择换另一种坦克（轻/中/重）。

## 4. 世界模型

### 4.1 地形

- **类型**：基于 Simplex/Perlin 噪声的高度图地形；
- **尺寸**：1024m × 1024m；
- **分辨率**：1 顶点/m，约 1M 顶点（通过 chunk 切片和 LOD 控制 drawcall）；
- **高度范围**：0 ~ 50m；
- **生成参数**：octaves = 4，frequency = 1/256，amplitude = 50m，种子由服务器启动时或重置时生成；
- **视觉**：硬边法线 + 基于高度/坡度的离散色带（低地草绿、中坡泥黄、高处岩灰），配合 Godot 4 的 TriPlanar 着色器实现方块感。

### 4.2 障碍物

障碍物由**噪声密度 + Poisson disk 采样**在地形上铺撒：

| 类型       | 数量（示例） | HP  | 碰撞形状         | 损毁表现                       |
|------------|-------------:|----:|------------------|--------------------------------|
| 小石块     | ~800         | 80  | 凸包             | 碎裂成 3–5 块方块碎片后消失     |
| 大石块     | ~150         | 250 | 凸包             | 同上，碎片更多                 |
| 树         | ~1200        | 120 | 胶囊（树干）     | 炮击点为铰链点向远离方向倒下   |
| 指挥部     | 2（每方 1）  | 2000| Box              | 大型爆炸粒子 + 塌陷动画        |

- **HP 不在 HUD 展示**（按原需求），但受击时有方块状贴花和碎屑粒子反馈；
- 损毁后永久从世界中消失，直到下一次世界重置；
- 障碍物在服务端以稀疏字典 `{obstacle_id: current_hp}` 管理，初始 HP 满值可由 id 查配置表推导，服务端只同步**与满值有差异的条目**（增量同步）。

### 4.3 世界生成流程（服务端）

1. 用 world seed 初始化 RNG；
2. 生成高度图（CPU 计算，存成 `Image` 后给 Jolt 生成 HeightFieldShape）；
3. 为每个阵营选定出生区：地图四角附近各一个阵营的 3 个出生区（共 6 个），约 80m × 80m，相对平坦；
4. 出生区外按噪声密度铺撒障碍物，避免和出生区/指挥部重叠；
5. 每个阵营基地中央放置指挥部；
6. 生成"世界快照"消息，发给新加入的客户端——只需 seed + obstacle diff（初始为空），客户端本地用相同算法重建。

### 4.4 客户端地形重建

客户端收到 seed 后**本地**跑相同的高度图和障碍物布撒算法。服务端**不**发送完整地形网格——节省带宽的关键。障碍物的初始摆放也由 seed + 同一算法还原；只有后续的破坏状态需要同步。

## 5. 坦克模型

### 5.1 三种坦克配置（待平衡）

| 属性              | 轻型 (Light) | 中型 (Medium) | 重型 (Heavy) |
|-------------------|------------:|--------------:|-------------:|
| 总 HP             | 600         | 900           | 1300         |
| 最大速度（km/h）  | 48          | 36            | 24           |
| 加速度 (m/s²)     | 4.5         | 3.0           | 2.0          |
| 炮塔转速 (°/s)    | 50          | 36            | 22           |
| 炮管仰俯角        | -5° ~ +18°  | -6° ~ +20°    | -7° ~ +22°   |
| 装填时间 (s)      | 3.0         | 4.5           | 6.5          |
| 炮弹初速 (m/s)    | 500         | 450           | 400          |
| 单发伤害          | 180         | 260           | 400          |
| 携弹量            | 30          | 24            | 18           |

*具体数值在实现阶段通过游戏内测试调整，本表为起始基准。*

### 5.2 部位化伤害模型（6 部位）

每辆坦克由一个根刚体 + 6 个子 Area3D（或 CollisionShape3D）组成，每个部位带独立 `part_hp`、`max_part_hp`、`multiplier`：

| 部位       | 子 HP 占比 | 伤害倍率 | 功能损坏后果                                                     |
|------------|-----------:|---------:|------------------------------------------------------------------|
| 车身 (Hull)| 40%        | 1.0×     | 归 0 → 整车爆炸（无论整车 HP 是否归 0）                          |
| 炮塔       | 15%        | 1.3×     | 归 0 → 无法瞄准（炮塔锁死）、无法开火，持续至重生                |
| 引擎       | 15%        | 1.5×     | 归 0 → 最高速度降至 25%、加速度减半；可"抢修" 30s 后恢复至 50%   |
| 左履带     | 10%        | 0.8×     | 归 0 → 无法左转/前进；"抢修" 20s 后恢复                          |
| 右履带     | 10%        | 0.8×     | 归 0 → 无法右转/前进；"抢修" 20s 后恢复                          |
| 顶部       | 10%        | 2.5×     | 归 0 → 直接爆炸（代表乘员阵亡）                                   |

**伤害计算：**
```
effective_damage = shell_damage × part_multiplier
part_hp -= effective_damage
total_hp -= effective_damage
```
爆炸条件（任一触发）：
- 整车 `total_hp <= 0`
- Hull 部位 HP <= 0
- 顶部 HP <= 0

**抢修机制**：玩家按 `R` 键并保持原地不动即可启动抢修，对**一处**失效部位恢复到 50% 子 HP：

- **可抢修**：履带（20s）、引擎（30s）；
- **不可抢修**：车身、炮塔、顶部（归 0 后需等死亡重生）；
- 抢修过程中如被击中则中断，需重新开始；
- 抢修完成后进入 60s 冷却，冷却期内不能再次抢修任何部位。

### 5.3 坦克移动

- **实现**：基于 Jolt Physics 的 RigidBody3D + 自定义地面附着逻辑（不直接用 VehicleBody3D——它更偏汽车；坦克的"差分驱动"用左右履带独立力矩建模更自然）；
- **输入 → 力**：
  - `W/S` → 两侧履带同向力（前进/后退）；
  - `A/D` → 两侧履带反向力（原地转向）；
  - 履带失效时对应侧力为 0；
  - 引擎损坏按比例缩放最大力；
- **物理**：有质量（轻 15t / 中 25t / 重 40t）、有惯性、有翻车可能（撞坡撞歪会侧翻，但有自动扶正机制避免卡死）；
- **爬坡限制**：大于 40° 坡爬不上去。

## 6. 战斗系统

### 6.1 炮弹（弹道）

- **模型**：抛物线 + 重力，无空气阻力、无风、无弹种（单种 AP 穿甲弹，未来可扩展）；
- **参数**：初速度按坦克型号（见 §5.1 表），重力 `g = 9.8 m/s²`；
- **实现**：服务端在 `FIRE` 事件中生成一个 `Shell` 实体，保存 `{origin, velocity, time_fired, owner_id}`。每 tick 前进：
  ```
  new_pos = origin + velocity * elapsed + 0.5 * g * elapsed²
  ```
- **命中检测**：服务端在每 tick 内对炮弹做**扫掠射线检测**（SweepTest），先与地形测，再与障碍物和坦克部位 Area3D 测；
- **客户端表现**：服务端 `FIRE` 事件广播后，每个客户端本地重放同样的弹道（不依赖持续同步炮弹位置，只在创建和命中时同步）。

### 6.2 命中判定与伤害

**服务端权威，带延迟补偿：**

1. 玩家按左键 → 客户端即时播放开火预览（枪口闪光、后坐动画），同时发 `FIRE` 请求（含 `client_tick`、炮管朝向）；
2. 服务端收到 `FIRE` 后：
   - 回溯到 `client_tick - interp_delay` 时刻（约 100ms）——此时刻对应客户端看到的世界状态；
   - 在该时刻的世界中模拟炮弹飞行（扫掠射线）；
   - 算出命中目标和具体部位；
3. 服务端广播 `SHELL_FIRED`（所有客户端放视觉弹道）和 `SHELL_HIT`（命中位置、受击者、部位、伤害）；
4. 受击者客户端播放命中反馈（火花、震动、部位高亮变红）。

### 6.3 瞄准镜视角（FPV）

- **触发**：按 `右键` 切入，再按一次切出；切入时炮塔转速降到正常的 70%（模拟"凑到瞄准镜前"的操作）；
- **UI**：军用光学标尺（见 §8.2）；
- **弹道预判**：系统**不**自动给出命中位置提示（硬核方向）——玩家自己看标尺上的弹道下坠刻度对准目标；
- **放大倍率**：默认 ×4，滚轮可切换 ×2 / ×4 / ×8。

### 6.4 队友识别

敌我识别通过**坦克颜色带**和**小地图圆点**；FPV 中瞄到友军时准星变灰，禁止开火（友伤关闭）。

## 7. 网络同步

### 7.1 Tick 与帧率

- **服务端**：20 Hz 固定 tick（每 tick 50ms）进行物理模拟和快照广播；
- **客户端**：60 Hz 渲染 + 20 Hz 输入发送；本地物理预测跟随服务端 tick 节奏。

### 7.2 权威性

**服务器权威**：所有影响游戏结果的判定（位置、伤害、命中、积分）由服务端裁决。客户端是"预测 + 显示层"。

### 7.3 客户端预测（自己的坦克）

1. 客户端每 50ms 向服务端发一次 `INPUT`（含 `tick`、WASD 状态、瞄准方向、开火按键）；
2. 客户端本地**立即**应用输入（用 `shared/tank/movement.gd` 跑一遍模拟），坦克即时响应；
3. 服务端回 `SNAPSHOT` 包含对该玩家 tick 的权威状态；
4. 客户端比较预测结果与权威结果：
   - 偏差 < 阈值（如 0.2m + 2°）：无动作（预测够准）；
   - 偏差 ≥ 阈值：以权威状态为准，**从该 tick 重放**此后已发送但未被 ack 的输入（rollback & reconcile）；
5. 视觉上插值到新位置，避免瞬移。

### 7.4 实体插值（其他玩家）

- 客户端为每个远程实体保留**最近 4 个快照**（约 200ms 历史）；
- 渲染时回溯 100ms 的时间戳，对两个邻近快照做线性 + 球面插值（位置线性、朝向 Slerp）；
- 保证显示流畅即使网络有抖动。

### 7.5 延迟补偿（射击）

服务端**为每个客户端**维护一个"近期快照环"（过去 1s，20 帧）：

- 收到 `FIRE` 时读 `client_tick` 并计算 `t = client_tick - 100ms`；
- 在 `t` 时刻的快照中查所有可能被击中的坦克位置；
- 重建该时刻坦克的 6 个部位 Collider 位置，对弹道做扫掠测试；
- 这样做的代价：每 FIRE 增加一次"回溯重建"计算，但仅发生在开火瞬间，成本可控。

### 7.6 快照压缩

- **Delta 编码**：每个快照基于客户端已 ack 的基准快照只发字段差异；
- **位打包**：位置用 16-bit 定点（精度 ±0.015m，范围 ±2048m）、朝向用 16-bit 四元数（smallest-three 压缩）；
- **AOI 粗略剔除**：虽然 1km × 1km 规模下不做完整 AOI，但一个坦克的 snapshot 字段分两层：全局（位置、朝向、总 HP）发给所有人；细节（炮塔角度、各部位子 HP、速度）只在 200m 范围内的客户端才收。

### 7.7 连接与断线

- **心跳**：服务端每 2s 发一次 `PING`，客户端回 `PONG`；30s 无响应视为掉线；
- **重连**：客户端断线后 60s 内用同一 session token 重连可恢复控制权和坦克；超时坦克自毁；
- **踢人**：掉线玩家坦克进入"AI 待机"（静止，但仍是可被攻击目标）直到重连或超时。

## 8. 客户端 UI

### 8.1 第三人称 HUD（信息丰富方案）

常驻元素：
- **左上**：小地图（110×110px），显示友军蓝点、敌军红点（仅目视/声纳范围内）、指挥部图标；
- **顶部中央**：世界剩余时间 / 红队积分 / 1000 / 蓝队积分；
- **右上**：本阵营玩家名单（名字 + 总 HP 颜色指示）；
- **屏幕中心**：自由瞄准准星（随炮塔方向）；
- **左下**：坦克剪影 + 4 个主要部位血条（Hull / Turret / L-Track / R-Track），颜色反映子 HP；
- **右下**：弹药类型 + 剩余数、装填进度条、当前速度。

临时元素：
- **击中反馈**：屏幕边缘红色脉冲（被击中方向）；
- **伤害数字**：小飘字显示自己对敌造成的伤害；
- **世界重置警告**：顶部红色横幅倒计时（最后 30s）；
- **部位损坏通知**：左下方一闪即逝的图标（如"引擎损坏"）。

### 8.2 第一人称瞄准镜 HUD（军用光学标尺方案）

- **中央**：黑色 vignette 四周遮罩，圆形视野区；
- **十字线**：水平 + 垂直细线；
- **水平刻度**：每 2 密位一短刻度，每 5 密位一长刻度（用于估计横向移动目标提前量）；
- **弹道下坠刻度**：竖直方向多条短横线，标注 400m / 600m / 800m 预期落点；
- **左上读数**：当前放大倍率（`×4`）+ 准星指向的地面距离（由服务端算，射线打到哪里）；
- **右上读数**：弹药类型和剩余数；
- **右下读数**：当前炮管仰角；
- **不显示**"系统帮你瞄哪里才命中"——保持硬核。

### 8.3 其他界面

- **加入/重生界面**：选坦克（轻/中/重）+ 选出生点（3 选 1），点"出击"；
- **计分板**（按 Tab）：两队玩家名字、击杀/死亡/助攻、ping；
- **设置**：灵敏度、音量、键位、画质、分辨率。

### 8.4 音效

- 方位化射击音（3D 定位）；
- 履带在不同地表（草 / 泥 / 石）有不同音色；
- 命中自己坦克时根据部位有差异化音效（金属撞击 / 装甲穿透 / 履带断裂）；
- 远距离爆炸音有延迟（按距离/340m 延后播放）。

## 10. 明确不做（Out of Scope）

以下**不在本 spec 范围内**，如需后续扩展另起 spec：

- 多弹种（HE / HEAT / APCR）、装甲穿深模拟；
- 侦察车 / 自行火炮等特种兵种；
- 匹配系统、多服务器路由、登录/账号系统；
- 持久化进度（经验值、解锁、皮肤）；
- 语音聊天；
- 手机端适配；
- AI 敌人；
- 地图编辑器；
- 大地图 AOI 分区（16–32 人规模下暂不需要）；
- 反作弊工程（客户端代码混淆、行为检测）。

## 11. 关键风险与缓解

| 风险                                         | 缓解                                                                       |
|----------------------------------------------|----------------------------------------------------------------------------|
| Web 端 WebAssembly + WebGL2 在低端机卡顿     | LOD、视锥剔除、障碍物静态合批；最低规格验证（Intel UHD 集显 + 4GB 内存）  |
| WebSocket over TCP 的抖动影响手感            | 预测 + 重放 + 插值；调优插值缓冲到 100–150ms；提供 ping 显示              |
| 客户端和服务端物理代码漂移                   | 共享 `shared/` 模块 + 单元测试在两端同样跑；关键函数加 assertion          |
| 三种坦克平衡困难                             | 数值集中在 `common/constants.gd` 配置表，支持热调整；内测小范围迭代       |
| 头部/履带等 hitbox 在方块模型上不直观         | hitbox 由设计师单独编辑的 `.tres` 资源控制，不绑死视觉模型                |
| 炮弹延迟补偿引发"我藏进掩体还被打到"的观感   | 设定 100ms 补偿上限；ping > 200 的客户端在 HUD 提示"网络不佳"             |

## 12. 实现优先级建议

供后续 writing-plans 参考的粗粒度阶段（细化留给实现规划）：

1. **骨架**：客户端 + 服务端能通过 WebSocket 交换消息；
2. **世界**：地形生成 + 静态障碍物可见；
3. **坦克 MVP**：单一坦克 + 第三人称视角 + 移动 + 开炮（hitscan 临时）+ 整车血量；
4. **弹道 + 部位伤害**：替换为抛物线 + 6 部位 + 功能损坏；
5. **联网稳健化**：客户端预测 + 插值 + 延迟补偿；
6. **可破坏环境**：石/树血量 + 损毁动画；
7. **FPV 瞄准镜**：军用标尺 UI + 切换逻辑；
8. **HUD 信息丰富版**：小地图、队友、比分、部位血条；
9. **3 种坦克 + 平衡迭代**；
10. **会话生命周期**：出生点选择、重生冷却、世界重置、积分；
11. **抢修、指挥部、完整音效**；
12. **集成测试、负载测试、打磨**。

## 13. 需要用户确认的默认值

我在用户没有明确表态但需要落地的地方填了合理默认值，以下是**需要用户审阅的关键默认**：

- **Tick rate**：服务端 20 Hz，客户端渲染 60 Hz；
- **世界尺寸**：1024m × 1024m（1km × 1km）；
- **重置时长**：2 小时；
- **积分阈值**：1000；
- **重生冷却**：10 秒；
- **出生点数**：每阵营 3 个；
- **3 种坦克的起始数值**（§5.1 表）；
- **6 部位的子 HP 占比和伤害倍率**（§5.2 表）；
- **延迟补偿上限**：100ms；
- **抢修机制**（履带 20s / 引擎 30s 施法、施法中受击中断、完成后 60s 冷却、恢复到 50%）—— 用户没提过，我加上是因为它让部位功能损坏不那么绝望；若不想要可删除；
- **指挥部**机制（给占点玩法一个落点）—— 同样是我补的，不想要可删；
- **单一弹种 AP**（明确延后多弹种）。
